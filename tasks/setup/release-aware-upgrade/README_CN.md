# 任务：发现新版本，一键升级 ClawMaster

**能力域**：Setup · **用时**：~5 min · **难度**：入门

> 当 ClawMaster 发布新版本时，应用会在顶部亮起一条橙色横幅。点一下就能打开设置页，看完版本更新说明后再一键下载安装包——不用自己去 GitHub 找。即使网络访问 GitHub 不畅也能检测到新版本，按平台自动挑对应安装包。

> 🌐 本任务以 **中文优先** 编写。English / 日本語 stubs：[README.md](./README.md) · [README_JP.md](./README_JP.md)

## 前置条件

1. 已完成 [wizard-ernie-glm](../wizard-ernie-glm/README_CN.md)，或任意方式跑起来 ClawMaster 桌面/Web 应用
2. 本机装的 ClawMaster 版本比最新 release 低（否则看到的是「已是最新版本」，没有升级流）

## TL;DR

1. 打开 ClawMaster，顶部出现橙色横幅：「**发现可用的 ClawMaster v{版本} 更新**」
2. 点「**查看更新**」→ 跳到设置页的「**ClawMaster 版本发布**」区块
3. 区块里会看到：当前版本、最新版本、版本更新说明
4. 点「**打开安装包**」下载对应本机系统的安装文件；或点「**版本更新日志**」去 GitHub 发布页查看完整变更
5. 关掉横幅后同一版本不会再打扰；只有下一个新版本才会再弹

## 第 1 步：等横幅自己出现

![](./images/01-banner.png)
*ClawMaster 仪表盘顶部会出现一条橙色横幅，提示新版本可用。旁边有「查看更新」按钮和一个关闭 ×。*

横幅是自动检查出来的：打开 ClawMaster 时，应用会对照一下本机版本和最新 release；如果发现更高的版本，就在顶部亮起横幅。如果你已经装的是最新版本，横幅不会出现。

按了右边的 × 就能关掉横幅。**同一版本** 的横幅关掉之后不会再弹，直到下一个新版本出现。

## 第 2 步：跳到设置页看详情

![](./images/02-release-section.png)
*点「查看更新」后落地到设置页的「ClawMaster 版本发布」区块：两张版本卡（当前 / 最新）、「↗ ClawMaster v{版本} 可以安装」提示，下方是版本更新说明（标题、要点列表、命令块）。*

横幅的「查看更新」按钮会带你到设置页的对应区块。在这里能一眼看全：

- **当前版本**：你机器上装的这一版
- **最新版本**：仓库里已发布的最新版
- **版本详情来源**：`来自 GitHub Releases` 或 `通过 npm 回退源检测` 两种标注
- **版本更新说明**：新版本的功能要点、升级提示、示例命令都会在这里直接渲染出来，不是一堆原始 Markdown 符号

读完再决定要不要升级。

## 第 3 步：一键下载适合本机的安装包

「打开安装包」按钮会根据你现在的操作系统，自动指向对应的资产：

- macOS → `.dmg`
- Windows → `.msi`
- Linux → `.AppImage`

所以你只要点一下就能拿到正确的安装文件，不用在 GitHub 发布页挑。旁边的「版本更新日志」按钮去 GitHub 看完整 changelog。

> 💡 ClawMaster 桌面端下载完成后，按正常的系统安装流程装一遍新版，再重新打开 ClawMaster。Web 端直接在命令行跑 `npm i -g clawmaster@latest` 然后重启 `clawmaster` 命令即可。

## 第 4 步：GitHub 访问不畅时怎么办

![](./images/04-npm-fallback.png)
*GitHub 访问不了时，区块依旧能检测到新版本（走 npm），但来源标注会切成「已通过 npm 回退源检测到版本」，版本更新说明换成一句兜底提示。点「打开发布页」可以换个网络再来看详情。*

如果你在国内网络环境、GitHub 访问比较慢或不通，应用会自动走 npm 这条备用链路。这样版本检测仍然能工作，横幅和版本号照常出现；只是详细的版本更新说明拿不到，得换个网络或走镜像再查。

需要镜像加速时，到「设置 → 能力 → npm 代理」打开镜像开关，填 `https://registry.npmmirror.com` 或你常用的镜像地址。设好之后重启一次 ClawMaster 或点区块右上角的「刷新」按钮即可。

## 对照：已是最新版本

![](./images/05-up-to-date.png)
*已是最新版本时：两个版本号相同，中间出现一个绿色的 ✓「ClawMaster 已是最新版本」。仪表盘上没有橙色横幅。*

如果你装的已经是最新版，顶部不会出现横幅，设置页区块也会用绿色 ✓ 告诉你「ClawMaster 已是最新版本」。

## 常见问题

**Q：装完新版本后，横幅却仍然显示升级到旧版本。** → 关闭 ClawMaster 再重新打开，让应用重读版本号。Web 端要重跑 `clawmaster` 命令。

**Q：版本更新说明显示成一串原始 Markdown 文字。** → 看区块里的来源标注，如果写的是「通过 npm 回退源」，说明当前取不到 GitHub 的详情。换个网络或配好镜像后点「刷新」。

**Q：点「打开安装包」没反应。** → 桌面端偶尔会被系统阻止外链跳转，可以用「版本更新日志」按钮跳转到 GitHub 发布页手动下载。浏览器端如果被弹窗拦截，允许弹窗后重试。

**Q：只想再看一次横幅。** → 重启应用后，只要有更新的版本号存在，横幅还会再出现一次；如果想立刻测试，可以在浏览器开发者工具里清掉本地存储再刷新。

**Q：镜像加速开关打开了，但是检测还是很慢。** → 后台走的是系统的 `npm` 命令行，部分版本会优先读你的全局 `.npmrc`。在终端跑 `npm config get registry` 看当前生效的 registry，必要时用 `npm config set registry=https://registry.npmmirror.com` 统一设置。
